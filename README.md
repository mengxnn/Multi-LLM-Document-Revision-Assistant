# 多 Agent 办公文档修订助手

这是一个本地运行的办公文档修订工具。你可以在网页里填写修改要求、配置大模型、生成修订稿，并继续基于历史版本修改。

## 快速启动

1. 解压项目压缩包。
2. 确认电脑已安装 Python 3.10 或更高版本。
3. 双击 `启动.bat`。
4. 等待脚本创建虚拟环境、安装依赖并打开网页。

默认网页地址：

```text
http://127.0.0.1:8765/
```

如果启动失败，不要关闭黑色窗口，先查看里面的错误信息。

## 第一次使用

打开网页后，先进入“模型”页。

1. 点击“添加配置”。
2. 填写：
   - 显示名称：给自己看的名称，如“阿里云 Qwen Plus”。
   - 模型ID：实际传给供应商的模型名，如 `qwen-plus`。
   - 供应商类型：可按供应商接口填写，如 `openai-compatible`。
   - 请求地址：供应商的 API endpoint。
   - API Key：供应商 API Key。
3. 如有需要，展开“高级设置”，调整超时时间、重试次数、搜索、视觉等能力。
4. 点击“保存配置”。
5. 在“现有配置”里点击“检测此配置”。
6. 检测通过后，分别点击“设为 writer”和“设为 reviewer”。

writer 负责生成或改写文档；reviewer 负责审查并给出下一轮修改建议。

## 新建项目

进入“新建”页。

1. 填写“修改要求”。这是必填项，可粘贴文字，也可上传一个或多个文件。
2. 可选填写“初稿”和“会议纪要”，每一类都支持同时上传多个文件。
3. 设置轮数。
4. 如只是测试流程，勾选 `dry-run`。
5. 点击“开始修订”。

上传多个文件时，建议用文件名区分内容，例如：以“初稿.docx”为主体，根据“数据补充.txt”更新第三节，并遵守“格式要求.md”。

粘贴的文字可以与上传文件同时使用。文件可分多次选择并追加到清单，选错时可在清单中单独删除。程序会先加入粘贴内容，再按清单顺序合并，并在项目中保留各个原文件。上传扫描版 PDF 时，需要勾选“启用 OCR”。

运行进度会显示在右侧。生成完成后，到“项目”页查看结果。

## 查看与继续修改

进入“项目”页。

- 点击“详情”查看项目版本、输入文件和产物路径。
- 展开后按钮会变成“折叠详情”，再次点击可收起。
- 点击“打开文件”可直接打开产物。
- 点击“文件位置”可打开所在目录。
- 如果要基于某个历史版本继续修改，先点击该版本的“基于此版继续”，再填写反馈并点击“继续修改”。
- 继续修改时，可分别选择是否保留新建项目时的修改要求、初稿和会议纪要。界面会显示各项字符数。
- 修改要求默认保留；初稿和会议纪要默认不保留，以减少不必要的模型输入。
- 本轮补充文件支持多次选择、追加和单独删除。扫描版 PDF 需要勾选“补充文件启用 OCR”。

所选基准版本和本轮反馈始终会传给模型。程序会在对应版本的运行记录中保存本轮材料选择，并保留补充文件原件。

## 模型配置管理

在“模型”页可以管理多个模型配置。

- “现有配置”：查看已保存配置。
- “添加配置”：新增配置。
- “编辑配置”：修改已有配置。
- “删除配置”：删除不再使用的配置。
- “设为 writer”：把该配置用于 writer。
- “设为 reviewer”：把该配置用于 reviewer。

如果删除的配置正在作为 writer 或 reviewer 使用，程序会自动取消对应激活状态。

## 输出位置

所有项目默认保存在：

```text
projects/
```

常见产物包括：

```text
final_draft/final.docx
final_draft/final.md
reviews/revision_summary.md
reviews/revision_summary.docx
final_review_report/final_review_report.md
final_review_report/final_review_report.docx
metadata/run_log.json
```

dry-run 输出在 `dry_run_outputs/`，真实模型输出在 `outputs/`。


## 常见问题

### 启动时提示找不到 Python

请先安装 Python 3.10 或更高版本，并确保安装时勾选“Add Python to PATH”。

### 第一次启动很慢

第一次会创建 `.venv` 虚拟环境并安装依赖，耗时取决于网络。之后启动会快很多。

### 连接检测正常，但运行失败

可能是模型供应商接口不稳定、免费模型限制、超时或上下文过长。可以尝试：

- 换一个模型配置。
- 增大超时时间。
- 减少运行轮数。
- 先用 `dry-run` 确认本地流程正常。

### 产物打不开或找不到

到“项目”页点击“详情”，使用“打开文件”或“文件位置”按钮查看。

### OCR 适合什么情况

OCR 只用于图片版 PDF 或扫描版 PDF 的文字识别。它可以把图片里的文字转成文本，但不能理解图片内容、图表含义或复杂版面。

文本型 PDF 会自动识别常见的单栏和双栏版面；检测到双栏时，程序会先读取左栏，再读取右栏。复杂表格、浮动图注和非常规多栏版面仍可能需要人工检查。

使用 OCR 前，需要先安装 Tesseract，并确认 `tesseract.exe` 已加入 PATH。程序也会自动识别 `tools\tesseract\tesseract.exe`、`C:\Program Files\Tesseract-OCR\tesseract.exe`，以及常见盘符下的 `Tesseract-OCR\tesseract.exe`。

可在网页“新建项目”中点击“检查 OCR”，查看实际使用路径、版本和已安装语言包。

如果要做成解压即用的压缩包，可以把 Tesseract 整个目录复制到项目的 `tools\tesseract\` 下，确保里面有 `tesseract.exe`。

如果安装在其他位置，可以在 `config/settings.env` 添加：

```text
TESSERACT_CMD=D:\Tesseract-OCR\tesseract.exe
```

扫描版双栏论文的 OCR 也会根据文字坐标调整阅读顺序，但准确性通常低于文本型 PDF。表格、脚注、浮动图注和非常规多栏版面仍建议先检查 OCR 文本。
